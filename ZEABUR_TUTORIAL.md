## 🎬 部署流程

### 步驟 1: 登入 Zeabur

```
1. 前往 https://zeabur.com/
2. 點擊右上角「登入」
3. 選擇「使用 GitHub 登入」
4. 授權 Zeabur 存取你的 GitHub
```

**畫面說明:**
```
┌────────────────────────────────────┐
│         Zeabur 首頁                │
│                                    │
│    🚀 Deploy in Seconds            │
│                                    │
│    [GitHub 登入] [開始使用]        │
└────────────────────────────────────┘
```

### 步驟 2: 建立專案

```
1. 登入後會看到 Dashboard
2. 點擊「Create Project」按鈕
3. 輸入專案名稱: telegram-ai-bot
4. 選擇區域: Taiwan (台灣)
5. 點擊「Create」
```

**畫面說明:**

### 步驟 3: 新增服務

```
1. 進入專案後點擊「Add Service」
2. 選擇「Git」選項
3. 如果是第一次,需要授權 GitHub

5. 點擊專案名稱
```

## 🔧 進階設定

### 持久化儲存設定

如果希望資料庫在重啟後保留:

```
步驟:
1. 在服務頁面點擊「Volumes」標籤
2. 點擊「Add Volume」
3. 設定:
   - 名稱: data
   - 掛載路徑: /app/data
4. 點擊「Create」
```

**畫面說明:**
```
┌────────────────────────────────────┐
│  持久化儲存                         │
│                                    │
│  名稱: [data              ]        │
│  路徑: [/app/data         ]        │
│  大小: [1 GB              ]        │
│                                    │
│             [建立]                  │
└────────────────────────────────────┘
```

### 設定自訂網域 (進階)

```
1. 點擊「Domains」標籤
2. 點擊「Add Domain」
3. 輸入你的網域
4. 設定 DNS 記錄
```

### 監控設定

```
1. 點擊「Metrics」標籤
2. 查看:
   - CPU 使用率
   - 記憶體使用量
   - 網路流量
   - 請求次數
```

## ❓ 常見問題與解決

### 問題 1: 建置失敗

**症狀:**
```
狀態: ❌ Build Failed
```

**解決方法:**
1. 檢查 Logs 找出錯誤
2. 確認 Dockerfile 語法正確
3. 確認 requirements.txt 完整
4. 點擊「Redeploy」重試

### 問題 2: 機器人無回應

**症狀:**
```
服務顯示 Running
但機器人不回應訊息
```

**解決方法:**
1. 檢查環境變數是否正確
2. 查看 Logs 找錯誤訊息
3. 確認 Token 有效
4. 重啟服務

### 問題 3: 記憶體超限

**症狀:**
```
狀態: 🔄 Restarting
日誌: Out of Memory
```

**解決方法:**
1. 升級到付費方案
2. 優化程式碼
3. 減少對話歷史
